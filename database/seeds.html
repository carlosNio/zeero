Seeds

Na Framework , Seeds são classes que registram dados actuais de uma tabela para posterior inserção ( semente ) . Também servem como gerador de dados estáticos para as tabelas após uma migração. Estas classes encontram-se no directório App/DataBase/Seeds e devem implementar dois metódos protegidos:

  • fields() - que retorna um array contendo os campos para a inserção.
  • data() - que retorna um array multidimensional contendo os dados de cada registro

Assim como um atributo protegido model que recebe o nome da classe Modelo a que os dados serão inseridos.


Exemplo de uma classe Seed 
                        
    class postSeed extends Seed
    {
        protected $model = 'post';

        protected function fields() 
        {
            return ['id_post', 'id_user', 'title', 'date'];
        }

        protected function data() 
        {
            return [
                ['1'  , '3'  , 'R for Data Science'  , '2022-07-12 20:22:33'],
                ['2'  , '11'  , 'JavaScript for Arduino'  , '2022-07-12 20:23:27'],
                ['3'  , '7'  , 'Ruby Programming'  , '2022-07-12 20:23:28'],
                ['4'  , '2'  , 'Cracking the code interview'  , '2022-07-12 20:24:18'],
                ['5'  , '4'  , 'Problem Solving'  , '2022-07-12 20:24:19'],
            ];
        }

    }
Gerando uma classe Seed via ZCLI

> php zcli make:seed [MODELO]

Após esse comando a classe será gerada e preenchida com os dados actuais da tabela.


Executado uma classe Seed via ZCLI
> php zcli db:seed [MODELO]

Executado uma classe Seed durante a migração via ZCLI
> php zcli migrate [NOME DA MIGRACAO] [ --up ou --down ] --m --s
> php zcli migrate:all [ --up ou --down ] --m --s
