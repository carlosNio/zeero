ERROR PAGE

Esta função é utilizada para apresentar uma página de erro apartir de qualquer ponto da aplicação. Ela recebe quatro argumentos obrigatórios, sendo um opcional.


A página padrão para erros ( error.twig ) encontra-se no directório App/Views na pasta renders . Para alterar a página de erro da aplicação basta adicionar o item (ERROR_PAGE) no arquivo .env com o valor que correponde ao nome de um arquivo no mesmo directório e pasta que a página padrão.


O último argumento (template) serve para apresentar uma página diferente da definida como padrão. É útil para páginas para erros específicos. 


          
    errorPage(string $title, string $message, int $code, string ?$template = null)
  • $title: o título a ser apresentado pela página.
  • $message: a mensagem no centro da página.
  • $code: o código HTTP da resposta.
  • $template: (opcional) o nome de um template alternativo ao padrão. 
    
    errorPage('Opps | NOT FOUND', 'Not Found', 404);

    errorPage('Opps | UNAUTHORIZED', 'UNAUTHORIZED', 401);
      
    errorPage('Opps | UNAUTHORIZED', 'UNAUTHORIZED', 401 , 'user-level-error.twig');
