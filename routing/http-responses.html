  • Resposta

    • Dependendo do despache de uma rota , o Router da Framework espera por um retorno.

  • Retornando um array

    • Ao retornar um array o Router dá uma resposta JSON ( Content-Type: application/json ) com o conteúdo do vector passado.

    Indíces do Array que serão interpretados pelo Router

    Chave Uso
    redirect O router faz um redirecionameto. ( string )
    headers Adiciona um ou mais cabeçalhos de resposta. ( array associativo )
    status define o código HTTP da resposta. ( valor inteiro )

    NOTA

    Se a chave redirect estiver definido no array, o router fará apenas o redirecionameto, setando o status: 302 .


  • Retornando um objecto

    • Ao retornar um objecto o Router dá uma resposta JSON ( Content-Type: application/json ) com o conteúdo do objecto .

  • Retornando uma string

    • Ao retornar uma string o Router dá uma resposta de texto simples ( Content-Type: text/html ) com a string retornada. 

        
        $app->get('/api/users' , function(){
              $users = User::all();
              return ['error' => false , 'list' => $users];
        });
          
         $app->get('/api/users' , function(){
             // ... 
            return ['status' => 400, 'headers' => [ 'X-Bad' => true ] , 'message' => 'bad request' ];
        });
